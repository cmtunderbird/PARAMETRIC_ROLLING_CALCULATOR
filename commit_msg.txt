fix #4: cache dashboard single-point weather fetch in App.jsx

fetchData() was calling the Open-Meteo APIs on every button press with no
caching, causing unnecessary network calls and contributing to 429 errors.

Fix:
- Import cacheGet/cacheSet from weatherCache.js
- Compute ptBounds (1-degree padding around the point) as cache key
- Check cache before each fetch (marine and atmo separately)
- Store result after a successful network fetch
- 1200ms inter-endpoint gap is skipped when atmo is served from cache

Cache TTLs (from weatherCache.js): marine = 6h, atmo = 3h.
Repeated clicks within those windows serve instantly from localStorage.
