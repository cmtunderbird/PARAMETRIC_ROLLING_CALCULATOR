fix: resolve CMEMS CORS by routing through Vite proxy

CORS issue: browser cannot reach nrt.cmems-du.eu/thredds/dodsC directly.
The browser treats it as a cross-origin request and blocks it.

Fix: Vite dev server proxy in vite.config.js
  /cmems-proxy/* -> https://nrt.cmems-du.eu/thredds/dodsC/*
  changeOrigin: true  (spoofs Host header)
  Authorization header relayed transparently by Vite http-proxy
  User-Agent set to identify as OPeNDAP client

This means all CMEMS requests are same-origin from the browser perspective.
No extra npm packages needed - Vite's built-in proxy handles everything.
Works automatically when launched via the desktop shortcut (npm run dev).
