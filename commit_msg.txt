feat: waypoint editor — edit, delete, insert, reorder, drag

Full in-panel waypoint editor with map interaction.

New state:
  editMode   — toggles the editor panel open/closed
  editingIdx — which row is in inline edit mode
  editForm   — {name, lat, lon} buffer for the row being edited

New helpers (all call mutateWps which re-numbers WPs, resets ETAs/weather):
  mutateWps(newWps)  — canonical mutation: renumber + setRoute + clear stale state
  wpDelete(idx)      — remove WP (blocked when only 2 remain)
  wpInsertAfter(idx) — insert midpoint WP after idx, auto-opens edit form
  wpMove(idx,lat,lon)— update position from map drag
  wpSaveEdit(idx)    — commit inline name/lat/lon edit with validation
  wpMoveUp(idx)      — swap with previous WP
  wpMoveDown(idx)    — swap with next WP

New component:
  DraggableWpMarker  — Leaflet Marker with draggable=true, fires onMove on dragend
  Rendered inside MapContainer when editMode is active (amber numbered circles)

UI panel (between Route Import and BOSP/EOSP):
  Collapsed: shows "✏ EDIT" button + WP count
  Expanded: scrollable list of all WPs, each row has:
    ▲ / ▼  — reorder up/down
    ✎      — open inline edit form (name + lat + lon fields, red border on invalid)
    +       — insert new WP at midpoint after this one
    ✕       — delete (disabled if only 2 WPs)
  Inline edit form: name, decimal-degree lat/lon, SAVE / cancel
  "+ ADD WAYPOINT AT END" button at bottom
  Hint: "Drag 🟡 markers on map to reposition"
  "✓ DONE" closes edit mode

Safety:
  mutateWps() enforces minimum 2 waypoints
  Coordinate validation: lat ∈ [-90,90], lon ∈ [-180,180]
  All mutations clear voyageWPs, voyageWeather, shipPos/Wx/Motion/MStat
  so stale data is never shown after route changes
