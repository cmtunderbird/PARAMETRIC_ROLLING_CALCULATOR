perf: replace slow dataset-open test with lightweight credential check

Previously handle_test() called open_dataset() + .values to fetch actual
wave data just to verify credentials — took 30-60s on first call.

Fix: use copernicusmarine.login(check_credentials_valid=True) which only
hits the auth server and returns a bool — no dataset open, no data download.

Measured: 3.3s Python auth + ~5s first-call worker overhead = ~8s total.
Was: 60+ seconds. 7-10x faster for the connection test.

Credentials are passed directly to login(), never written to disk
(force_overwrite=False ensures ~/.copernicusmarine is not modified).
