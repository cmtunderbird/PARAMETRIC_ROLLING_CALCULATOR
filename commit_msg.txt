fix: restore missing totalH in scrubber + fix find().times pattern

Two bugs introduced during scrubber editing:

1. const totalH = maxHourIdx was accidentally removed from the forecast
   scrubber IIFE. dayTicks used totalH (line 675) but it was undefined,
   causing the day tick marks on the timeline to be blank/NaN.
   Restored: const totalH = maxHourIdx after baseMs.

2. marineGrid.results.find(r=>r.times)?.[0] — this finds the first result
   object that has a .times array, then indexes [0] on the object itself
   (not the array), always returning undefined. The truthy check that
   followed was therefore always false, so baseMs always fell back to nowMs.
   Fixed: find(r => r.times?.length > 0) then use .times[0] directly.
   Same fix applied to the ship-scrubber IIFE.
