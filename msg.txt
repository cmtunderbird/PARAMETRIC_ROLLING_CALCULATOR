fix: retry serverAlive() 10x/15s for post-reboot startup race + spawn error logging

Root cause of persistent error after system restart:
serverAlive() was called once with a single 5-second window. After reboot,
cmems-server.js takes 3-8 seconds to start (Node.js boot + Python worker
init + copernicusmarine import). The React app loads in ~500ms, so clicking
TEST CONNECTION immediately after the window appears always hits the server
before it's ready — returning false and showing the error every time.

src/cmemsProvider.js:
  - serverAlive() now retries up to 10 times with 1.5s delay between attempts
    (15s total budget). Returns true as soon as the server responds 200.
    First attempt is instant — no delay added for users who wait before testing.
  - Updated error message: clearer "failed to start" vs "not running" wording.

electron/main.js:
  - Log the exact spawn command: "[cmems] spawning: <binary> <script>"
    Immediately shows whether node was found and what path is used.
  - Added 'error' event handler on cmemsServer to catch ENOENT (node not
    in PATH) and other spawn failures with clear console output.
